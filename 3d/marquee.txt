Component 10: Marquee (Magic UI)
What it does: Infinite horizontal scroll of cards/content. Two rows moving in opposite directions. Pauses on hover. Gradient fade on edges. CSS --duration variable controls speed.

Source: Magic UI (@/registry/magicui/marquee)

Dependencies: Same as NumberTicker — need the actual Marquee source component. Magic UI marquee is typically pure CSS animation (translateX) with a React wrapper. Zero heavy deps.

Where it fits in Boot Force:

Primary use — Announcements ticker on homepage:

Single row marquee scrolling active announcements
Each item: announcement title + brief content
Replaces our planned basic ticker with something visually polished
Secondary uses:

Tech stack showcase — a marquee of technology logos/icons the club works with (React, Python, Figma, etc.). Adds visual richness to the homepage or about section.
Member testimonials / highlights — the demo shows review cards. We could use this for member quotes about the club experience. Low priority but visually impressive if time allows.
Modifications needed for our brand:

Card styling: replace bg-gray-950/[.01] with our glass style: bg-[#140A29]/40 backdrop-blur-md border-[#2D1B4E]
Hover: border glow border-[#00F0FF] shadow-[0_0_15px_rgba(0,240,255,0.3)]
Text colors: #F2F0FF for names, #A39BBA for secondary text
Edge gradients: from-[#0A0514] instead of from-background to match our void color
For announcements: simplify to just text content, no avatar/username — single row, not two
pauseOnHover stays — good UX
What NOT to do with it:

Don't use it for the leaderboard (leaderboard needs real-time updates, not a scroll loop)
Don't put two marquee rows on the homepage (one is elegant, two is busy)
Assessment: Keeper. Clean, performant, versatile.
