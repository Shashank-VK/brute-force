# Database Designer

Use this skill to plan database structure with a clear model, constraints, and migration path.

## When to Use
- Designing a new schema.
- Reviewing relationships, normalization, or indexing.
- Writing or adjusting migration scripts.

## Procedure
1. Identify the entities, relationships, and access patterns.
2. Define keys, constraints, indexes, and cardinality.
3. Check normalization and denormalization tradeoffs.
4. Map the model to migrations and query usage.
5. Review edge cases such as deletes, backfills, and scale.
