# Webapp Testing

Use this skill to verify that a web app behaves correctly in the browser and to capture evidence of failures.

## When to Use
- Testing a new or changed user flow.
- Debugging a UI bug that needs browser reproduction.
- Capturing screenshots, traces, or console evidence.

## Procedure
1. Start the app in the same environment the browser test will use.
2. Reproduce the user flow with stable selectors and minimal assumptions.
3. Capture screenshots or traces around failures.
4. Check the observed behavior against the intended outcome.
5. Report concise findings with the exact step that failed.
